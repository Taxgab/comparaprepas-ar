## 🧵 HILO 2: IA Local y Privacy-First

### Tweet 1/7
```
Tu IA no debería vivir en la nube de otro. ☁️❌

Correr modelos localmente no es paranoia. Es soberanía digital.

Cuando tu IA procesa tus datos en servidores ajenos:
• Perdés privacidad
• Dependés de su uptime
• Pagás por lo que es tuyo

Hay otra forma. 👇
```

### Tweet 2/7
```
¿Qué significa "IA local" hoy?

🖥️ Modelos que corren en TU hardware (GPU, NPU, CPU)
📁 Datos que nunca salen de tu infraestructura
🔐 Keys de API que no compartís con nadie
⚡ Latencia que no depende de su servidor

Vos tenés el control. Punto.
```

### Tweet 3/7
```
Hardware accesible HOY:

• MacBook M1/M2/M3: 7B-13B params nativo
• RTX 4090: 70B params quantized
• Jetson Orin: edge AI embebida
• Raspberry Pi 5: modelos tiny para IoT

No necesitás un datacenter. Necesitás elegir bien el modelo.
```

### Tweet 4/7
```
Casos donde local es OBLIGATORIO:

🏥 Salud: historiales médicos
⚖️ Legal: contratos y casos
🏦 Finanzas: datos de clientes
🔒 Seguridad: vulnerabilidades internas
👥 RRHH: evaluaciones de empleados

Si los datos son sensibles, la nube no es opción.
```

### Tweet 5/7
```
El trade-off que nadie admite:

Nube: más potente, menos privado, costo recurrente
Local: menos potente, más control, costo upfront

Para 80% de los casos de uso empresariales, local gana.

El 20% restante justifica híbrido.
```
